<div id="readability-page-1" class="page"><div data-v-36ca7acf=""><h2 id="getting-started" tabindex="-1">Getting Started <a href="#getting-started" aria-label="Permalink to &quot;Getting Started&quot;">​</a></h2><h2 id="overview" tabindex="-1">Overview <a href="#overview" aria-label="Permalink to &quot;Overview&quot;">​</a></h2><p>Vitest (pronounced as <em>"veetest"</em>) is a next generation testing framework powered by Vite.</p><p>You can learn more about the rationale behind the project in the <a href="/guide/why">Why Vitest</a> section.</p><h2 id="trying-vitest-online" tabindex="-1">Trying Vitest Online <a href="#trying-vitest-online" aria-label="Permalink to &quot;Trying Vitest Online&quot;">​</a></h2><p>You can try Vitest online on <a href="https://vitest.new/" target="_blank" rel="noreferrer">StackBlitz</a>. It runs Vitest directly in the browser, and it is almost identical to the local setup but doesn't require installing anything on your machine.</p><h2 id="adding-vitest-to-your-project" tabindex="-1">Adding Vitest to Your Project <a href="#adding-vitest-to-your-project" aria-label="Permalink to &quot;Adding Vitest to Your Project&quot;">​</a></h2><a href="https://vueschool.io/lessons/how-to-install-vitest?friend=vueuse" target="_blank" bg-green:10="" px4="" py3="" rounded="" block="" mt2="" flex="" items-center="" gap2=""><!--[-->Learn how to install by Video<!--]--></a><div><p><label data-title="npm" for="tab-O1Hz7Y_">npm</label><label data-title="yarn" for="tab-9TEe1io">yarn</label><label data-title="pnpm" for="tab-87-bspn">pnpm</label><label data-title="bun" for="tab-BnQHSed">bun</label></p><div><p><span>bash</span></p><pre tabindex="0"><code><span><span>npm</span><span> install</span><span> -D</span><span> vitest</span></span></code></pre></div></div><div><p>TIP</p><p>Vitest requires Vite &gt;=v5.0.0 and Node &gt;=v18.0.0</p></div><p>It is recommended that you install a copy of <code>vitest</code> in your <code>package.json</code>, using one of the methods listed above. However, if you would prefer to run <code>vitest</code> directly, you can use <code>npx vitest</code> (the <code>npx</code> tool comes with npm and Node.js).</p><p>The <code>npx</code> tool will execute the specified command. By default, <code>npx</code> will first check if the command exists in the local project's binaries. If it is not found there, <code>npx</code> will look in the system's <code>$PATH</code> and execute it if found. If the command is not found in either location, <code>npx</code> will install it in a temporary location prior to execution.</p><h2 id="writing-tests" tabindex="-1">Writing Tests <a href="#writing-tests" aria-label="Permalink to &quot;Writing Tests&quot;">​</a></h2><p>As an example, we will write a simple test that verifies the output of a function that adds two numbers.</p><div><p><span data-title="sum.js">sum.js</span></p><div><p><span>js</span></p><pre tabindex="0"><code><span><span>export</span><span> function</span><span> sum</span><span>(</span><span>a</span><span>, </span><span>b</span><span>) {</span></span>
<span><span>  return</span><span> a </span><span>+</span><span> b</span></span>
<span><span>}</span></span></code></pre></div></div><div><p><span data-title="sum.test.js">sum.test.js</span></p><div><p><span>js</span></p><pre tabindex="0"><code><span><span>import</span><span> { expect, test } </span><span>from</span><span> 'vitest'</span></span>
<span><span>import</span><span> { sum } </span><span>from</span><span> './sum.js'</span></span>
<span></span>
<span><span>test</span><span>(</span><span>'adds 1 + 2 to equal 3'</span><span>, () </span><span>=&gt;</span><span> {</span></span>
<span><span>  expect</span><span>(</span><span>sum</span><span>(</span><span>1</span><span>, </span><span>2</span><span>)).</span><span>toBe</span><span>(</span><span>3</span><span>)</span></span>
<span><span>})</span></span></code></pre></div></div><div><p>TIP</p><p>By default, tests must contain <code>.test.</code> or <code>.spec.</code> in their file name.</p></div><p>Next, in order to execute the test, add the following section to your <code>package.json</code>:</p><div><p><span data-title="package.json">package.json</span></p><div><p><span>json</span></p><pre tabindex="0"><code><span><span>{</span></span>
<span><span>  "scripts"</span><span>: {</span></span>
<span><span>    "test"</span><span>: </span><span>"vitest"</span></span>
<span><span>  }</span></span>
<span><span>}</span></span></code></pre></div></div><p>Finally, run <code>npm run test</code>, <code>yarn test</code> or <code>pnpm test</code>, depending on your package manager, and Vitest will print this message:</p><div><p><span>txt</span></p><pre tabindex="0"><code><span><span>✓ sum.test.js (1)</span></span>
<span><span>  ✓ adds 1 + 2 to equal 3</span></span>
<span><span></span></span>
<span><span>Test Files  1 passed (1)</span></span>
<span><span>     Tests  1 passed (1)</span></span>
<span><span>  Start at  02:15:44</span></span>
<span><span>  Duration  311ms</span></span></code></pre></div><div><p>WARNING</p><p>If you are using Bun as your package manager, make sure to use <code>bun run test</code> command instead of <code>bun test</code>, otherwise Bun will run its own test runner.</p></div><p>Learn more about the usage of Vitest, see the <a href="https://vitest.dev/api/" target="_blank" rel="noreferrer">API</a> section.</p><h2 id="configuring-vitest" tabindex="-1">Configuring Vitest <a href="#configuring-vitest" aria-label="Permalink to &quot;Configuring Vitest&quot;">​</a></h2><p>One of the main advantages of Vitest is its unified configuration with Vite. If present, <code>vitest</code> will read your root <code>vite.config.ts</code> to match with the plugins and setup as your Vite app. For example, your Vite <a href="https://vitejs.dev/config/shared-options.html#resolve-alias" target="_blank" rel="noreferrer">resolve.alias</a> and <a href="https://vitejs.dev/guide/using-plugins" rel="noreferrer" target="_blank">plugins</a> configuration will work out-of-the-box. If you want a different configuration during testing, you can:</p><ul><li>Create <code>vitest.config.ts</code>, which will have the higher priority</li><li>Pass <code>--config</code> option to CLI, e.g. <code>vitest --config ./path/to/vitest.config.ts</code></li><li>Use <code>process.env.VITEST</code> or <code>mode</code> property on <code>defineConfig</code> (will be set to <code>test</code> if not overridden) to conditionally apply different configuration in <code>vite.config.ts</code></li></ul><p>Vitest supports the same extensions for your configuration file as Vite does: <code>.js</code>, <code>.mjs</code>, <code>.cjs</code>, <code>.ts</code>, <code>.cts</code>, <code>.mts</code>. Vitest does not support <code>.json</code> extension.</p><p>If you are not using Vite as your build tool, you can configure Vitest using the <code>test</code> property in your config file:</p><div><p><span data-title="vitest.config.ts">vitest.config.ts</span></p><div><p><span>ts</span></p><pre tabindex="0"><code><span><span>import</span><span> { defineConfig } </span><span>from</span><span> 'vitest/config'</span></span>
<span></span>
<span><span>export</span><span> default</span><span> defineConfig</span><span>({</span></span>
<span><span>  test: {</span></span>
<span><span>    // ...</span></span>
<span><span>  },</span></span>
<span><span>})</span></span></code></pre></div></div><div><p>TIP</p><p>Even if you do not use Vite yourself, Vitest relies heavily on it for its transformation pipeline. For that reason, you can also configure any property described in <a href="https://vitejs.dev/config/" target="_blank" rel="noreferrer">Vite documentation</a>.</p></div><p>If you are already using Vite, add <code>test</code> property in your Vite config. You'll also need to add a reference to Vitest types using a <a href="https://www.typescriptlang.org/docs/handbook/triple-slash-directives.html#-reference-types-" target="_blank" rel="noreferrer">triple slash directive</a> at the top of your config file.</p><div><p><span data-title="vite.config.ts">vite.config.ts</span></p><div><p><span>ts</span></p><pre tabindex="0"><code><span><span>/// &lt;</span><span>reference</span><span> types</span><span>=</span><span>"vitest"</span><span> /&gt;</span></span>
<span><span>import</span><span> { defineConfig } </span><span>from</span><span> 'vite'</span></span>
<span></span>
<span><span>export</span><span> default</span><span> defineConfig</span><span>({</span></span>
<span><span>  test: {</span></span>
<span><span>    // ...</span></span>
<span><span>  },</span></span>
<span><span>})</span></span></code></pre></div></div><p>The <code>&lt;reference types="vitest" /&gt;</code> will stop working in Vitest 3, but you can start migrating to <code>vitest/config</code> in Vitest 2.1:</p><div><p><span data-title="vite.config.ts">vite.config.ts</span></p><div><p><span>ts</span></p><pre tabindex="0"><code><span><span>/// &lt;</span><span>reference</span><span> types</span><span>=</span><span>"vitest/config"</span><span> /&gt;</span></span>
<span><span>import</span><span> { defineConfig } </span><span>from</span><span> 'vite'</span></span>
<span></span>
<span><span>export</span><span> default</span><span> defineConfig</span><span>({</span></span>
<span><span>  test: {</span></span>
<span><span>    // ... Specify options here.</span></span>
<span><span>  },</span></span>
<span><span>})</span></span></code></pre></div></div><p>See the list of config options in the <a href="./../config/">Config Reference</a></p><div><p>WARNING</p><p>If you decide to have two separate config files for Vite and Vitest, make sure to define the same Vite options in your Vitest config file since it will override your Vite file, not extend it. You can also use <code>mergeConfig</code> method from <code>vite</code> or <code>vitest/config</code> entries to merge Vite config with Vitest config:</p><div><p><label data-title="vitest.config.mjs" for="tab-rThfJiF">vitest.config.mjs</label><label data-title="vite.config.mjs" for="tab-O2RfvU7">vite.config.mjs</label></p><div><div><p><span>ts</span></p><pre tabindex="0"><code><span><span>import</span><span> { defineConfig, mergeConfig } </span><span>from</span><span> 'vitest/config'</span></span>
<span><span>import</span><span> viteConfig </span><span>from</span><span> './vite.config.mjs'</span></span>
<span></span>
<span><span>export</span><span> default</span><span> mergeConfig</span><span>(viteConfig, </span><span>defineConfig</span><span>({</span></span>
<span><span>  test: {</span></span>
<span><span>    // ...</span></span>
<span><span>  },</span></span>
<span><span>}))</span></span></code></pre></div><div><p><span>ts</span></p><pre tabindex="0"><code><span><span>import</span><span> { defineConfig } </span><span>from</span><span> 'vite'</span></span>
<span><span>import</span><span> Vue </span><span>from</span><span> '@vitejs/plugin-vue'</span></span>
<span></span>
<span><span>export</span><span> default</span><span> defineConfig</span><span>({</span></span>
<span><span>  plugins: [</span><span>Vue</span><span>()],</span></span>
<span><span>})</span></span></code></pre></div><p>However, we recommend using the same file for both Vite and Vitest, instead of creating two separate files.</p></div></div></div><h2 id="workspaces-support" tabindex="-1">Workspaces Support <a href="#workspaces-support" aria-label="Permalink to &quot;Workspaces Support&quot;">​</a></h2><p>Run different project configurations inside the same project with <a href="/guide/workspace">Vitest Workspaces</a>. You can define a list of files and folders that define your workspace in <code>vitest.workspace</code> file. The file supports <code>js</code>/<code>ts</code>/<code>json</code> extensions. This feature works great with monorepo setups.</p><div><p><span data-title="vitest.workspace.ts">vitest.workspace.ts</span></p><div><p><span>ts</span></p><pre tabindex="0"><code><span><span>import</span><span> { defineWorkspace } </span><span>from</span><span> 'vitest/config'</span></span>
<span></span>
<span><span>export</span><span> default</span><span> defineWorkspace</span><span>([</span></span>
<span><span>  // you can use a list of glob patterns to define your workspaces</span></span>
<span><span>  // Vitest expects a list of config files</span></span>
<span><span>  // or directories where there is a config file</span></span>
<span><span>  'packages/*'</span><span>,</span></span>
<span><span>  'tests/*/vitest.config.{e2e,unit}.ts'</span><span>,</span></span>
<span><span>  // you can even run the same tests,</span></span>
<span><span>  // but with different configs in the same "vitest" process</span></span>
<span><span>  {</span></span>
<span><span>    test: {</span></span>
<span><span>      name: </span><span>'happy-dom'</span><span>,</span></span>
<span><span>      root: </span><span>'./shared_tests'</span><span>,</span></span>
<span><span>      environment: </span><span>'happy-dom'</span><span>,</span></span>
<span><span>      setupFiles: [</span><span>'./setup.happy-dom.ts'</span><span>],</span></span>
<span><span>    },</span></span>
<span><span>  },</span></span>
<span><span>  {</span></span>
<span><span>    test: {</span></span>
<span><span>      name: </span><span>'node'</span><span>,</span></span>
<span><span>      root: </span><span>'./shared_tests'</span><span>,</span></span>
<span><span>      environment: </span><span>'node'</span><span>,</span></span>
<span><span>      setupFiles: [</span><span>'./setup.node.ts'</span><span>],</span></span>
<span><span>    },</span></span>
<span><span>  },</span></span>
<span><span>])</span></span></code></pre></div></div><h2 id="command-line-interface" tabindex="-1">Command Line Interface <a href="#command-line-interface" aria-label="Permalink to &quot;Command Line Interface&quot;">​</a></h2><p>In a project where Vitest is installed, you can use the <code>vitest</code> binary in your npm scripts, or run it directly with <code>npx vitest</code>.
